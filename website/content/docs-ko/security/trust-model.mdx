# 신뢰 모델

OpenKakao를 쓴다는 것은, 이미 KakaoTalk 데스크톱 앱이 의존하는 환경에 접근할 수 있는 로컬 CLI를 신뢰한다는 뜻입니다. 그 신뢰는 명시적이어야 합니다.

## CLI가 의존하는 것

OpenKakao는 다음에 의존합니다.

- 로그인된 KakaoTalk macOS 앱
- 로그인 흐름에 필요한 로컬 앱 캐시와 요청 메타데이터
- 자격 증명 저장을 위한 로컬 파일시스템
- 계정 및 메시징 작업에 필요한 Kakao REST/LOCO 엔드포인트

## 사용자가 신뢰하게 되는 동작

CLI를 실행할 때 사용자는 다음을 신뢰하게 됩니다.

- 인증과 동작에 필요한 로컬 데이터만 읽을 것
- 자격 증명을 제한된 권한으로 로컬 저장할 것
- 지원 작업에 필요한 Kakao 엔드포인트에만 연결할 것
- 기본적으로 민감한 값을 로그나 쉘 출력에 흘리지 않을 것

## 신뢰하지 않아도 되는 것

사용자는 다음을 신뢰하도록 요구받지 않습니다.

- OpenKakao 호스팅 백엔드
- 프로젝트가 운영하는 원격 자격 증명 릴레이
- 메시지나 계정에 대한 불투명한 텔레메트리
- 추출된 비밀정보의 기본 클라우드 저장

의도된 신뢰 경계는 로컬 기기와 Kakao 사이이지, 로컬 기기와 OpenKakao 서비스 사이가 아닙니다.

## 주요 리스크 범주

### 자격 증명 처리

CLI는 민감한 인증 정보를 다룰 수밖에 없습니다. 따라서 어떤 기기에서 실행하느냐가 중요합니다.

### 계정 행동

어떤 흐름이 동작한다고 해서 저위험이라는 뜻은 아닙니다. 고빈도 전송, 수상한 패턴, 오픈채팅 자동화는 제한 가능성을 높일 수 있습니다.

### 업스트림 불안정성

역공학 기반 동작은 앱이나 서버 변경 이후 깨질 수 있습니다. 이건 예외가 아니라 운영 리스크입니다.

## 권장 신뢰 태도

OpenKakao는 권한이 큰 로컬 운영자 도구처럼 써야 합니다.

- 통제 가능한 기기에서만 사용할 것
- 자동화 범위를 좁게 유지할 것
- 공격적인 아웃바운드보다 읽기 중심 워크플로를 우선할 것
- unattended job에 넣기 전에 명령을 직접 검토할 것
- 계정 안전은 결국 사용자 자신의 절제에 달려 있다고 가정할 것
