# 제한사항

OpenKakao는 의도적으로 좁은 도구입니다. 경계를 무시하면 brittle automation이나 불필요한 계정 리스크를 얻게 됩니다.

## 플랫폼 제한

- macOS 전용
- KakaoTalk 데스크톱 앱이 설치되어 있고 로그인돼 있어야 함
- 일부 흐름은 로컬 앱 캐시와 런타임 상태에 의존함
- 소스 빌드에는 최신 Rust toolchain이 필요함

## 제품 제한

- 비공식이며 역공학 기반
- Kakao 서버나 클라이언트 변경이 계속 호환된다는 보장이 없음
- 다중 사용자 SaaS나 account farming 용도로 설계되지 않음
- 지원되는 엔터프라이즈 통합을 대체하지 않음

## 리스크 제한

- 자격 증명 추출은 정의상 민감한 작업임
- outbound automation은 과하면 의심스러워 보일 수 있음
- 오픈채팅 활동은 개인 채팅보다 제재 리스크가 큼
- 기술적으로 가능하더라도 잦고 부자연스러운 행동 패턴은 좋지 않음

## 운영 제한

- upstream 동작이 바뀌면 토큰 갱신이 실패할 수 있음
- watch 모드는 장시간 네트워크 안정성에 의존함
- 프로토콜 문서는 현재 역공학 결과 기준이며 시간이 지나면 낡을 수 있음
- 미디어/업로드 흐름은 텍스트 메시징과 별도로 바뀔 수 있음

## 문서의 태도

이 문서는 제한사항을 edge case로 취급하지 않습니다. 제한사항도 운영 모델의 일부입니다. 강한 호환성 보장, 계정 안전 보장, 공식 지원이 필요하다면 OpenKakao는 맞지 않는 선택이라고 보는 편이 맞습니다.
