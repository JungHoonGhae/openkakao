# 자동화 개요

OpenKakao는 하나의 거대한 플랫폼처럼 다루기보다, 로컬 building block으로 다룰 때 가장 유용합니다.

## 잘 맞는 사용처

- unread summary와 개인 대시보드
- 로컬 검색/저장을 위한 채팅 export 파이프라인
- `watch` 기반 event-driven alert
- 통제된 send step이 필요한 운영자 도구
- 로컬에서 메시지를 요약하거나 라우팅하는 LLM / agent workflow

## 두 가지 자동화 스타일

### Pull

지연을 감수할 수 있고 더 단순한 실패 모델을 원하면 scheduled read를 사용하세요.

```bash
openkakao-rs unread --json
openkakao-rs loco-read <chat_id> -n 50 --json
```

### Push

event-driven 동작이 필요하면 `watch`를 사용하세요.

```bash
openkakao-rs watch --chat-id <chat_id>
```

`watch`는 더 강력하지만, 그만큼 재연결 로직, 관측성, 안전한 fallback의 중요성이 커집니다.

## 기본 설계 규칙

자동 응답 루프보다, 읽고 요약하고 분류하는 작업을 먼저 두세요. outbound automation은 첫 단계가 아니라 마지막 단계여야 합니다.

## 흔한 도구 조합

- JSON 필터링과 shaping에는 `jq`
- 로컬 저장에는 `sqlite3`
- 스케줄링에는 cron 또는 launchd
- 오케스트레이션에는 셸 스크립트
- 요약과 triage에는 로컬 또는 원격 LLM 도구
