# 설정

OpenKakao는 운영 정책을 다음 위치에 저장할 수 있습니다.

```text
~/.config/openkakao/config.toml
```

같은 정책 플래그를 매번 반복하고 싶지 않은 unattended 서비스, launchd job, cron task에 적합합니다.

## 우선순위

설정은 다음 순서로 해석됩니다.

1. CLI flags
2. `config.toml`
3. 내장 안전 기본값

## 예시

```toml
[mode]
unattended = true

[send]
allow_non_interactive = true
default_prefix = true

[watch]
allow_side_effects = true
default_max_reconnect = 10

[auth]
prefer_relogin = true
auto_renew = true
```

## 현재 필드

| Key | Type | Purpose |
|-----|------|---------|
| `mode.unattended` | bool | 이 환경이 interactive 확인 없이 실행될 수 있음을 선언 |
| `send.allow_non_interactive` | bool | `send -y`, `send-file -y`, `send-photo -y` 허용 |
| `send.default_prefix` | bool | 기본 outgoing prefix 유지 여부 |
| `watch.allow_side_effects` | bool | watch 모드의 읽음 처리, 로컬 hook, webhook 허용 |
| `watch.default_max_reconnect` | int | `watch` 기본 재연결 횟수 재정의 |
| `auth.prefer_relogin` | bool | 복구 시 relogin과 renew 중 무엇을 먼저 시도할지 결정 |
| `auth.auto_renew` | bool | 복구 중 refresh-token renewal을 허용하거나 건너뜀 |

## Auth 정책 예시

### 보수적 기본값

```toml
[auth]
prefer_relogin = true
auto_renew = true
```

복구 순서는 다음과 같습니다.

1. saved credentials
2. relogin
3. renew
4. Cache.db extraction

### Renewal 생략

```toml
[auth]
prefer_relogin = true
auto_renew = false
```

renewal이 시끄럽거나 신뢰하기 어려울 때, relogin과 fresh extraction만 남기고 싶다면 이 구성이 맞습니다.

### Renewal 우선 시도

```toml
[auth]
prefer_relogin = false
auto_renew = true
```

현재 환경에서 refresh-token renewal이 더 싼 복구 경로라고 판단할 근거가 있을 때만 사용하세요.

## 여전히 명령줄에 남겨둘 것

다음 값들은 계속 명령줄에 두는 편이 맞습니다.

- open chat 작업의 `--force`
- `--webhook-url` 같은 구체적인 목적지
- `--hook-chat-id` 같은 matcher

정책은 지속돼야 하고, 대상은 실행 중인 명령에 드러나 있어야 합니다.
