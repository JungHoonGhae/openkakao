## read (REST)

REST API로 채팅방 메시지를 읽습니다.

```bash
openkakao-rs read <chat_id> [OPTIONS]
```

| Flag | 설명 |
|------|------|
| `-n <COUNT>` | 메시지 수 (기본값: 30) |
| `--before <LOG_ID>` | 해당 logId 이전 (역방향 페이지네이션) |
| `--cursor <LOG_ID>` | cursor에서 이어서 읽기 |
| `--since <DATE>` | 특정 날짜 이후 필터 (YYYY-MM-DD) |
| `--all` | 가능한 모든 메시지 가져오기 |

> [!WARNING]
> REST는 최근에 열린 채팅의 메시지만 돌려줍니다. 전체 히스토리가 필요하면 `loco-read`를 사용하세요.

`read`는 다음 상황에 맞습니다.

- 캐시를 빠르게 확인할 때
- 짧고 가벼운 읽기
- KakaoTalk 앱 상태가 Pilsner 캐시를 실제로 채웠는지 확인하는 디버깅

## loco-read (LOCO)

LOCO `SYNCMSG` 프로토콜로 메시지 히스토리를 읽습니다. **메시지 읽기에서는 이 명령을 권장합니다.**

```bash
openkakao-rs loco-read <chat_id> [OPTIONS]
```

| Flag | 설명 |
|------|------|
| `-n <COUNT>` | 메시지 수 (기본값: 30) |
| `--all` | 서버에 남아 있는 전체 히스토리 가져오기 |
| `--since <DATE>` | 특정 날짜 이후 필터 (YYYY-MM-DD) |
| `--cursor <LOG_ID>` | cursor에서 이어서 읽기 |
| `--delay-ms <MS>` | 배치 간 지연 (기본값: 100) |
| `--force` | 오픈채팅 읽기 허용 |
| `--json` | JSON 출력 |

```bash
# 전체 히스토리를 JSON으로 저장
openkakao-rs loco-read <chat_id> --all --json > history.json
```

`loco-read`는 다음 상황에 맞습니다.

- 실제 export
- 자동화 파이프라인
- 더 믿을 수 있는 히스토리 수집
- GUI에서 최근에 채팅을 열었는지에 의존하면 안 되는 작업

## search

채팅방에서 메시지를 검색합니다.

```bash
openkakao-rs search <chat_id> "keyword"
```

## export

메시지를 파일로 내보냅니다.

```bash
openkakao-rs export <chat_id> --format json
openkakao-rs export <chat_id> --format csv -o messages.csv
openkakao-rs export <chat_id> --format txt -o chat.txt
```

| Flag | 설명 |
|------|------|
| `--format <FORMAT>` | `json`, `csv`, `txt` |
| `-o <FILE>` | 출력 파일 경로 |
