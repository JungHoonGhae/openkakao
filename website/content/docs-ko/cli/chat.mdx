## chats (REST)

Pilsner REST API로 채팅방 목록을 조회합니다.

```bash
openkakao-rs chats [OPTIONS]
```

| Flag | 설명 |
|------|------|
| `-a, --all` | 모든 채팅방 표시 (페이지네이션) |
| `-u, --unread` | 읽지 않은 채팅만 표시 |
| `--search <QUERY>` | 제목으로 검색 |
| `--type <TYPE>` | `dm`, `group`, `memo`, `open` 필터 |

> [!NOTE]
> REST는 KakaoTalk 앱에서 최근에 연 채팅만 돌려줍니다. 전체 접근이 필요하면 `loco-chats`를 사용하세요.

다음 상황에 적합합니다.

- 빠른 unread/account 인접 뷰
- 낮은 세션 오버헤드
- Pilsner 캐시에 이미 들어 있는 데이터만으로 충분한 경우

## loco-chats (LOCO)

LOCO 프로토콜로 전체 채팅방 목록을 조회합니다.

```bash
openkakao-rs loco-chats [--all]
```

| Flag | 설명 |
|------|------|
| `--all` | PlusChat 등 모든 채팅 타입 포함 |

다음 상황에 적합합니다.

- 기준이 되는 전체 채팅 목록이 필요할 때
- 다른 LOCO 채팅 작업과 같은 동작을 원할 때
- `loco-read`, `send`, `watch`와 일관성을 맞추고 싶을 때

## loco-chatinfo

채팅방의 상세 정보를 가져옵니다.

```bash
openkakao-rs loco-chatinfo <chat_id>
```

메타데이터, 설정, 참여자 수가 포함된 raw LOCO `CHATONROOM` 응답을 반환합니다.

`chat_id`에 `0`을 넣으면 나와의 채팅(MemoChat)을 찾거나 생성할 수 있습니다.
