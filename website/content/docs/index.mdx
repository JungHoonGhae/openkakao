# OpenKakao

OpenKakao is an unofficial KakaoTalk CLI for macOS aimed at developers, terminal-native users, and automation-heavy workflows. It gives you access to personal chats through reverse-engineered KakaoTalk app behavior, not an official public API.

> [!WARNING]
> OpenKakao is powerful, but it is not risk-free. Read the [Trust Model](/docs/security/trust-model), [Data & Credentials](/docs/security/data-and-credentials), and [Safe Usage](/docs/security/safe-usage) pages before using it on a real account.

## What It Can Do

| Capability | Why it matters |
|---|---|
| Read chats and message history | Pull recent or full conversation history into scripts and local tools |
| Send text, files, and media | Build controlled outbound automations and operator workflows |
| Watch real-time events | Trigger scripts, alerts, or summaries when messages arrive |
| Export JSON | Pipe KakaoTalk data into `jq`, `sqlite`, `llm`, or other Unix tools |
| Inspect friends and profile data | Build personal workflow tools around your account graph |

## Why the docs start with trust

The first question for this project is usually not what it can do. It is whether you should trust it with a real KakaoTalk account. These docs are structured to answer that directly:

1. What the CLI reads from your machine
2. What credentials it keeps locally
3. What network endpoints it talks to
4. What behavior has higher account risk
5. How to keep usage narrow, auditable, and reversible
