## Introduction

These commands answer a simple question: do you want a cheap cache-backed read, or do you need the real chat history surface?

<Cards>
  <Card icon={<Server />} title="read (REST)">
    Fast, lightweight, and limited to recently populated chat cache.
  </Card>
  <Card icon={<History />} title="loco-read (LOCO)">
    The reliable path for history, exports, and automation pipelines.
  </Card>
  <Card icon={<Search />} title="search">
    Keyword lookup within a chat room.
  </Card>
  <Card icon={<Database />} title="export">
    Persist message history into JSON, CSV, or text files.
  </Card>
</Cards>

<Callout title="Default to loco-read when the result matters">
  `read` is useful for quick inspection. `loco-read` is the safer default when you are building anything persistent, repeatable, or automation-heavy.
</Callout>

## read (REST)

Read messages from a chat room via REST API.

```bash
openkakao-rs read <chat_id> [OPTIONS]
```

| Flag | Description |
|------|-------------|
| `-n <COUNT>` | Number of messages (default: 30) |
| `--before <LOG_ID>` | Before this logId (backward pagination) |
| `--cursor <LOG_ID>` | Resume from cursor |
| `--since <DATE>` | Filter after date (YYYY-MM-DD) |
| `--all` | Fetch all available messages |

<Callout type="warn" title="REST is cache-backed">
  REST only returns messages for recently opened chats. Use `loco-read` for full history or anything that should not depend on GUI recency.
</Callout>

Use `read` for:

- quick cache inspection
- short, lightweight reads
- debugging whether KakaoTalk app state has populated the Pilsner cache

## loco-read (LOCO)

Read message history via LOCO SYNCMSG protocol. **This is the recommended command for reading messages.**

```bash
openkakao-rs loco-read <chat_id> [OPTIONS]
```

| Flag | Description |
|------|-------------|
| `-n <COUNT>` | Number of messages (default: 30) |
| `--all` | Fetch complete server-retained history |
| `--since <DATE>` | Filter after date (YYYY-MM-DD) |
| `--cursor <LOG_ID>` | Resume from cursor |
| `--delay-ms <MS>` | Delay between batches (default: 100) |
| `--force` | Allow reading open chats |
| `--json` | JSON output |

```bash
# Full history as JSON
openkakao-rs loco-read <chat_id> --all --json > history.json
```

Use `loco-read` for:

- real exports
- automation pipelines
- reliable history fetches
- anything that should not depend on whether you opened the chat in the GUI recently

<Callout title="Think of loco-read as the real read path">
  If you are deciding between convenience and correctness, choose `loco-read` first and optimize later only if the extra surface is unnecessary.
</Callout>

## search

Search messages in a chat room.

```bash
openkakao-rs search <chat_id> "keyword"
```

Use this when you know the chat and need message lookup, not full export flow.

## export

Export messages to a file.

```bash
openkakao-rs export <chat_id> --format json
openkakao-rs export <chat_id> --format csv -o messages.csv
openkakao-rs export <chat_id> --format txt -o chat.txt
```

| Flag | Description |
|------|-------------|
| `--format <FORMAT>` | `json`, `csv`, or `txt` |
| `-o <FILE>` | Output file path |
